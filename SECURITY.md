# Security Policy

## Scope

This repository covers the LocalLens Chrome extension product, its public support and privacy site, and the release helpers used to generate store and launch collateral.

Report issues here when they affect:

- extension behavior or packaged assets in `extension/`
- public support or privacy content served from `public_site/`
- release metadata or scripts that could leak sensitive data or misdirect users

## Reporting a vulnerability

Use GitHub private vulnerability reporting for this repository when it is available.

If private reporting is not enabled, do not open a public issue for a live security problem. Use the support contact path linked from the LocalLens public site and include enough detail to reproduce the issue safely.

Include:

- affected surface and version or commit
- reproduction steps
- expected impact
- whether the issue exposes user content, tokens, or linkable browsing data

## Response expectations

- Acknowledge receipt as soon as practical.
- Ask for clarification only when needed to reproduce safely.
- Coordinate a fix before public disclosure when the report is valid.

## Out of scope

- purely stylistic documentation issues
- feature requests without a security impact
- reports that depend on modifying a local developer machine outside the documented workflow
