## Deployment Rules

1. Set GitHub description, homepage, and topics explicitly.
2. Keep support, privacy-policy, and reviewer-instructions links pointed at the public site base generated by this repo. Use `LOCALLENS_PUBLIC_SITE_BASE` when publishing the Cloudflare Worker surface.
3. Do not mix LocalLens product assets with OpenClaw operator skills.
4. Keep generated `dist/` outputs out of version control unless the user explicitly asks for checked-in release artifacts.
5. Regenerate store assets and release notes from this repo before every tagged release.
6. Do not cancel or replace a Chrome Web Store draft that is already pending review unless the change fixes a real acceptance blocker for that submitted version.

## Default Verification

1. Run `python3 -m pytest -q`.
2. Run `npm test`.
3. If packaging or release metadata changed, run `python3 scripts/build_extension_zip.py --extension-dir extension --out /tmp/locallens-extension.zip`, `python3 scripts/generate_launch_manifest.py --repo-root . --out /tmp/locallens-launch-manifest.json`, and `python3 scripts/generate_listing_copy.py --repo-root . --manifest /tmp/locallens-launch-manifest.json --out /tmp/locallens-store-listing.json`.
4. If `codex_harness` is available locally, run `python3 -m codex_harness audit . --strict --min-score 90`.
5. Before pushing release or reviewer-collateral changes, run the reviewer gate with `.githooks/pre-push` or explain why the gate could not run.
